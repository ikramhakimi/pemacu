---
name: build-page-from-design-system
description: Use this when creating a new page that must follow the existing design system, reuse current components, and avoid unauthorized new UI patterns.
---

When building a new page:
1. Inspect existing related pages, sections, and components first.
2. List the existing patterns/components to reuse.
3. Build the page using approved current patterns.
4. Do not create new visual patterns unless explicitly requested.
5. If the current system is insufficient, propose options before implementation.
